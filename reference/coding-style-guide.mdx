## 1. Mongoose Schema Structure

### Basic Structure

- Use strict schema definition
- Include timestamps (`createdAt`, `updatedAt`)
- Use `ObjectId` for references
- Define validation rules
- Use virtual fields

### Schema Definition

```javascript
const mongoose = require('mongoose');
const { Schema } = mongoose;

const ExampleSchema = new Schema({
  name: { type: String, required: true },
  email: { type: String, unique: true },
  userId: { type: Schema.Types.ObjectId, ref: 'User' },
  createdAt: { type: Date, default: Date.now },
  updatedAt: { type: Date, default: Date.now }
});
```

### Validation

- Use enums for status
- Use required fields
- Use unique constraints
- Add custom validators

### Indexes

- Add indexes for queries
- Use compound indexes
- Consider sparse indexes

## 2. Variable Naming Conventions

- Use **camelCase** for variable names (e.g., `firstName`, `userEmail`)
- Use **PascalCase** for class names (e.g., `UserModel`, `CampaignController`)
- Use **UPPER\_SNAKE\_CASE** for constants (e.g., `MAX_RETRIES`, `API_KEY`)
- Use **snake\_case** for database fields and environment variables (e.g., `first_name`, `DATABASE_URL`)
- Be descriptive but concise
- Avoid single-letter variable names except for simple loops
- Use plural for arrays (e.g., `users`, `tasks`)
- Prefix boolean variables with `is`, `has`, or `can` (e.g., `isActive`, `hasError`)

## 3. File Structure and Organization

- Use clear, descriptive file names (e.g., `agent.controller.js`, `campaign.controller.js`)
- Group related functionality into modules (e.g., `agent/`, `campaign/`, `hubspot/`)
- Use consistent directory structure:

```text
api/
├── agent/
│   ├── agent.controller.js
│   ├── agent.model.js
│   └── agent.helper.js
├── campaign/
└── ...
```

## 4. Code Style

- Use strict mode (`'use strict'`)
- Use arrow functions for consistency
- Use async/await for asynchronous operations
- Use single quotes for strings
- Use ES6\+ syntax (destructuring, template literals, etc.)

## 5. Function Definitions

```javascript
exports.functionName = async (req, res) => {
  try {
    // business logic
    return sendResponse(res, SUCCESS, 'Success message', data);
  } catch (error) {
    return errReturned(res, error);
  }
};
```

## 6. Error Handling

- Use try-catch blocks for error handling
- Use standardized error response functions (`errReturned`)
- Use proper error codes (`BADREQUEST`, `SUCCESS`, etc.)
- Log errors appropriately

## 7. Response Patterns

```javascript
// Success response
return sendResponse(res, SUCCESS, 'Success message', data);

// Error response
return errReturned(res, error);
```
